# PRD · 小丑牌 Web V1.2.0 · Joker 系统

**Version** V1.2.0 · 2026-05-16  
**基于** V1.0.0 核心计分引擎 + V1.1.0 视觉升级

---

## 1. 一句话定位

在 V1.0.0 单局比牌型的基础上，引入小丑牌（Joker）、多关卡闯关、商店与金币系统，让游戏从"打一局算分"升级为"边闯关边构筑牌组"的 Roguelite 体验——面向已上手 V1.0.0 的玩家，体感更接近原版 Balatro。

---

## 2. 本轮范围 / 不做的事

### 做（V1.2.0 MVP）

- 3 关闯关模式（目标分 300 / 800 / 2000，关间进商店）
- 8 张 Joker 牌库 + 3 个 Joker 装备槽位
- 出牌后 Joker 触发结算（按槽位顺序叠加倍率）
- 金币系统（每次出牌获得金币）
- 商店（关卡结束后弹出，售卖 3 张随机 Joker）
- 出牌结算 UI 反馈（触发 Joker 金边闪烁 + 浮字）
- 修复 V1.0.0 的 `isStraight` P0 Bug（K-Q-J-10-9 等顺子无法识别）

### 不做（明确排除，见第 11 章）

- Joker 出售 / 升级 / 稀有度
- 塔罗牌、星球牌、增益 / Debuff 体系
- Boss 关卡 / 盲注难度
- 商店刷新（花金币换商品）
- 音效 / 粒子特效
- 移动端适配 / 多人联机

## 5. Joker 系统详细规则

### 5.1 8 张 Joker 完整表格

| # | 名称 | 类型 | 价格 | 触发条件 | 效果（精确） | 展示文案 |
|---|------|------|------|----------|-------------|----------|
| 1 | 小丑（Joker） | 无条件 | $4 | 每次出牌必触发 | 倍率 +4 | "每次出牌 +4 倍率" |
| 2 | 欢乐小丑 | 牌型条件 | $4 | 本次出牌包含对子（含两对、葫芦、四条） | 倍率 +8 | "出牌含对子时 +8 倍率" |
| 3 | 狂热小丑 | 牌型条件 | $5 | 本次出牌包含三条（含葫芦、四条） | 倍率 +12 | "出牌含三条时 +12 倍率" |
| 4 | 贪婪小丑 | 花色条件 | $5 | 出的牌中每张方块（♦）牌 | 每张方块 倍率 +4 | "出的每张方块牌 +4 倍率" |
| 5 | 色欲小丑 | 花色条件 | $5 | 出的牌中每张红心（♥）牌 | 每张红心 倍率 +4 | "出的每张红心牌 +4 倍率" |
| 6 | 暴怒小丑 | 花色条件 | $5 | 出的牌中每张黑桃（♠）牌 | 每张黑桃 倍率 +4 | "出的每张黑桃牌 +4 倍率" |
| 7 | 卑鄙小丑 | 数量条件 | $5 | 出的牌中每张梅花（♣）牌 | 每张梅花 倍率 +4 | "出的每张梅花牌 +4 倍率" |
| 8 | 半个小丑 | 数量条件 | $6 | 本次出牌张数 ≤ 3 张 | 倍率 +20 | "出牌 ≤3 张时 +20 倍率" |

> **牌型条件判定说明**：
> - "欢乐小丑"触发条件：识别出的牌型为 对子 / 两对 / 葫芦 / 四条 之一（即含有至少一个 pair）
> - "狂热小丑"触发条件：识别出的牌型为 三条 / 葫芦 / 四条 之一（即含有至少一个 triple）
> - 以上两张可同时触发（例如葫芦会同时触发欢乐 +8 和狂热 +12）

### 5.2 槽位规则

- **槽位数量**：玩家固定拥有 3 个 Joker 槽位（Slot 1 / Slot 2 / Slot 3）
- **初始状态**：3 个槽位全部为空
- **装备规则**：在商店购买 Joker 后，自动填入最左侧的空槽位；若 3 个槽位全满，则无法购买（商店购买按钮禁用，提示"槽位已满"）
- **替换 / 卖出**：V1.2.0 不支持替换和卖出，Joker 一旦装备不可移除
- **结算顺序**：出牌后按 Slot 1 → Slot 2 → Slot 3 的顺序依次检查并应用 Joker 效果，全部为加法累加到 mult（倍率）

### 5.3 出牌结算流程（新版计算公式）

```
Step 1  识别牌型，查表取得 base_chips（基础分）和 base_mult（基础倍率）

Step 2  计算打出牌的点数之和 card_chips
        A=11, K/Q/J=10, 2-10 按面值

Step 3  合并基础芯片
        total_chips = base_chips + card_chips

Step 4  依次遍历 Slot 1 → Slot 2 → Slot 3，对每张已装备 Joker：
        检查触发条件（牌型 / 花色 / 数量）
        若满足 → mult += joker.mult_bonus（累加倍率）
        若不满足 → 跳过

Step 5  计算最终得分
        round_score = total_chips × (base_mult + accumulated_joker_mult)

Step 6  accumulated_score += round_score
        检查是否达到本关目标
```

**公式速写**：

```
最终得分 = (base_chips + Σ card_chips) × (base_mult + Σ triggered_joker_mult)
```

**计算示例**：

> 第 2 关，装备了「小丑」(+4) 和「色欲小丑」(每张红心+4)  
> 打出：♥A ♥K ♥Q ♥J ♥10 → 皇家同花顺  
> base_chips = 100，base_mult = 8，card_chips = 11+10+10+10+10 = 51  
> Slot1 小丑：无条件，mult += 4  
> Slot2 色欲小丑：5 张均为红心，mult += 4×5 = 20  
> 最终得分 = (100 + 51) × (8 + 4 + 20) = 151 × 32 = **4832 分**

## 6. 金币系统

### 获得规则

- **每次出牌后**（无论得分高低）自动获得金币
- 公式：`获得金币 = $3 + 剩余出牌次数 × $1`
- 弃牌不获得金币

**金币获得示例**（每关共 4 次出牌机会）：

| 第几次出牌 | 出牌前剩余次数 | 出牌后剩余次数 | 获得金币 |
|------------|---------------|---------------|----------|
| 第 1 次 | 4 | 3 | $3 + 3×$1 = **$6** |
| 第 2 次 | 3 | 2 | $3 + 2×$1 = **$5** |
| 第 3 次 | 2 | 1 | $3 + 1×$1 = **$4** |
| 第 4 次 | 1 | 0 | $3 + 0×$1 = **$3** |

> 一局最多可获得 $6+$5+$4+$3 = **$18**，实际因过关可能不打满 4 次。

### 初始金币

- 游戏开始时（第 1 关开始前）玩家拥有 **$4**
- 第 2 关、第 3 关开始时，金币为上一关结束进商店后的剩余金币（跨关卡保留）

### 消费规则

- 金币仅在商店中消费，购买 Joker
- 金币不足时不可购买（按钮禁用，提示"金币不足"）
- 没有其他消费途径

## 7. 商店规则

### 出现时机

每关目标达成（过关）后，弹出商店页面（覆盖游戏界面）。  
第 3 关通关后**不弹商店**，直接显示通关结算弹窗。

### 商品内容

- 固定展示 **3 张随机 Joker**，从 8 张牌库中随机抽取（可重复出现）
- 每张 Joker 显示：名称 / 效果描述 / 价格 / 购买按钮

### 购买逻辑

- 点击"购买"：扣除对应金币，Joker 装入最左侧空槽位
- 已装备同名 Joker：仍可再次购买（同张可重复装备，但受槽位上限限制）
- 槽位已满（3/3）：所有购买按钮禁用，显示提示"Joker 槽位已满"
- 金币不足：对应购买按钮禁用，显示提示"金币不足"

### 刷新机制

- V1.2.0 **不支持刷新**，进入商店后商品固定，不可花钱换新

### 跳过逻辑

- 商店右上角或底部有"继续"按钮
- 点击"继续"关闭商店，进入下一关（手牌和牌堆重新发牌，Joker 和金币保留）

## 8. 页面 & 交互改动

### 8.1 主游戏页 HUD 变化

- **金币显示**：HUD 顶栏右侧，格式 `$ 8`，颜色金黄色（`#f5c842`）
- **关卡显示**：HUD 顶栏左侧，格式 `第 2 关 / 共 3 关`

### 8.2 Joker 槽位区

位置：手牌区上方，操作栏下方（或左侧竖排，视实现空间而定）

布局：3 个固定槽位卡片横排，每个槽位显示：
- 空状态：虚线边框 + 灰色"空槽"文字
- 已装备：Joker 名称 + 效果简述（1 行）+ 正常边框

### 8.3 出牌结算 UI 反馈

每次出牌后，若触发 Joker：

1. **金色边框闪烁**：被触发的 Joker 卡片边框变为金色（`#f5c842`），闪烁动画 0.4s（亮-暗-亮），闪烁 2 次
2. **浮字动画**：Joker 卡片旁边从中心向上浮出文字，格式 `+X Mult`（例如 `+4 Mult`），动画时长 0.8s，透明度从 1 渐变到 0，同时向上移动 24px
3. 多张 Joker 同时触发时，各自独立播放上述动画，无需串行

## 9. 状态字段补充

在 V1.0.0 state 基础上，新增以下字段：

```js
// 新增字段（在原 gameState 对象中追加）
{
  // 关卡
  "currentLevel":   number,   // 当前关卡 1-3
  "levelTargets":   [300, 800, 2000],  // 各关目标分（固定）

  // 金币
  "coins":          number,   // 当前金币数，初始 4

  // Joker 槽位
  "jokerSlots":     Joker[],  // 长度固定为 3，空槽为 null
  // 例：[{id:"joker", name:"小丑", multBonus:4}, null, null]

  // 商店
  "shopItems":      Joker[],  // 当前商店展示的 3 张 Joker（随机生成）
  "shopVisible":    boolean,  // 商店是否显示

  // 游戏状态扩展（原 gameStatus 值扩展）
  // "playing" | "levelClear" | "win" | "lose"
}

// Joker 数据结构
{
  "id":          string,   // 唯一标识，如 "joker" / "jolly" / "zany" 等
  "name":        string,   // 显示名称
  "type":        string,   // "unconditional" | "hand_type" | "suit" | "count"
  "price":       number,   // 商店售价
  "multBonus":   number,   // 固定加成的倍率值（花色条件为每张的倍率，代码循环）
  "condition":   string,   // 触发条件描述（用于 UI 展示）
  "description": string    // 效果文案（用于 UI 展示）
}
```

> V1.0.0 中 `target: 300` 字段改为读取 `levelTargets[currentLevel - 1]`，不再写死。

## 11. 不做什么

- **Joker 出售**：玩家不能卖出已装备的 Joker 换回金币（增加复杂度，留 v1.3.0 考虑）
- **Joker 升级**：Joker 无升级路径，买到即固定效果
- **Joker 稀有度**：无普通/稀有/传奇区分，商店等概率随机
- **塔罗牌 / 星球牌**：两类增益道具完全不做
- **Boss 关卡**：第 3 关不附加特殊规则，纯靠目标分制造压力
- **盲注难度**：无小盲/大盲/Boss 盲注体系
- **商店刷新**：进入商店后 3 张商品固定，不可花金币刷新
- **动画音效**：仅做 CSS 动画（金边闪烁/浮字），不引入音频文件
- **移动端适配**：V1.2.0 仍然是桌面 Web，不做响应式

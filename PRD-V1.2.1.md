## 1. 一句话定位

手牌发出后自动按点数从大到小排列，相同点数按花色聚合，让玩家一眼看出可组的牌型，消除 8 张牌中手动扫视的认知负担。

---

## 2. 本轮范围

### 做（V1.2.1 MVP）

- 手牌自动排序（发牌 / 补牌 / 弃牌补牌 / 重新开始 全部时机）
- 手牌区顶部新增「排序切换」小按钮（按点数 ↔ 按花色）
- 记住玩家最后选择的排序模式，本局内持久（刷新页面不需要持久化）

### 不做（明确排除，见第 8 节）

- 拖拽手动排序
- 自动选牌 / 自动推荐牌型
- 本局结束后跨局持久化排序偏好

## 3. 问题描述

**当前现象**：`buildDeck()` 构建后 `shuffle()` 随机化整副牌，发出的 8 张手牌顺序完全随机，相同点数或花色的牌散落各处。

**用户场景**：玩家拿到 8 张手牌，要在 1-2 秒内判断能凑出什么牌型（对子？顺子？同花？）。随机排列要求玩家依次扫视 8 张牌，心算位置，体验差。

**期望**：手牌落定后自动重排，视觉上相同点数的牌紧挨在一起，玩家扫一眼就能看出「这里有两张 K、那里有三张 5」。原版 Balatro 即采用此策略。

---

## 4. 核心方案

### 4.1 默认排序规则（按点数）

**主键**：点数从大到小

| 点数 | 序号（rank） |
|------|-------------|
| A    | 14 |
| K    | 13 |
| Q    | 12 |
| J    | 11 |
| 10   | 10 |
| 9–2  | 9–2 |

**次键**：相同点数时按花色排，优先级 ♠ > ♥ > ♦ > ♣

> 花色优先级依据：♠ 在扑克中视觉最重（常见最高花色），♥ 其次，♦ 第三，♣ 最低。与主流扑克惯例一致，玩家无需学习。

**实际序列示例**：

```
手牌原始：2♣ K♥ 5♦ K♠ A♦ 5♣ Q♠ A♠
排序结果：A♠ A♦ K♠ K♥ Q♠ 5♦ 5♣ 2♣
```

**花色序号映射**（代码用）：

```js
const SUIT_RANK = { '♠': 3, '♥': 2, '♦': 1, '♣': 0 };
```

### 4.2 备选排序规则（按花色）

**主键**：花色优先级 ♠ > ♥ > ♦ > ♣

**次键**：相同花色内，点数从大到小

**实际序列示例**：

```
手牌原始：2♣ K♥ 5♦ K♠ A♦ 5♣ Q♠ A♠
排序结果：A♠ K♠ Q♠  K♥  A♦ 5♦  5♣ 2♣
         ——黑桃——  ─红心─  ──方块──  ─梅花─
```

### 4.3 触发时机

以下四个时机调用 `sortHand()`，**覆盖所有手牌更新场景**：

| 时机 | 说明 |
|------|------|
| 开局发牌 | `buildDeck()` + 初始发 8 张后 |
| 出牌后补牌 | `playHand()` 结算完成，从牌堆补足 8 张后 |
| 弃牌后补牌 | `discardHand()` 完成，从牌堆补足 8 张后 |
| 重新开始 | `resetGame()` 后发牌完成时 |

> **原则**：手牌 DOM 只通过 `render()` 统一刷新，因此在 `render()` 内部、渲染手牌节点前调用 `sortHand()` 最安全，无需在多处埋点。

### 4.4 排序切换按钮

**推荐**：提供「按点数 / 按花色」切换按钮。

**推荐理由**：
- 凑顺子时「按点数」最直观；凑同花时「按花色」才能一眼看出同花张数。
- 原版 Balatro 有此功能，目标用户对标 Balatro 玩家，切换习惯已养成。
- 实现成本极低（一个 `state.sortMode` 字段 + 一次 `render()` 调用），收益高。

**默认值**：`sortMode = 'rank'`（按点数）

## 5. UI 改动

切换按钮放在**手牌区正上方，居左**，与「出牌 / 弃牌」操作按钮同排或紧邻一行：

```
┌──────────────────────────────────────────────┐
│  [按点数 ▼]  [按花色]          出牌  弃牌    │  ← 操作栏
├──────────────────────────────────────────────┤
│  A♠  A♦  K♠  K♥  Q♠  5♦  5♣  2♣            │  ← 手牌区（已排序）
└──────────────────────────────────────────────┘
```

**按钮样式**：

- 两个按钮并排，宽度均等（约 70px），小号字体（12px）
- 当前激活的模式：实色背景（`--color-accent` 橙色）+ 白色文字
- 未激活的模式：透明背景 + 边框 + 灰色文字
- 点击后立即触发 `render()`，手牌重排，无需动画

## 6. 技术实现要点

### 新增函数 `sortHand()`

```js
function sortHand(hand, mode = state.sortMode) {
  const SUIT_RANK = { '♠': 3, '♥': 2, '♦': 1, '♣': 0 };
  return [...hand].sort((a, b) => {
    if (mode === 'rank') {
      // 主键：点数降序；次键：花色降序
      return b.rank - a.rank || SUIT_RANK[b.suit] - SUIT_RANK[a.suit];
    } else {
      // 主键：花色降序；次键：点数降序
      return SUIT_RANK[b.suit] - SUIT_RANK[a.suit] || b.rank - a.rank;
    }
  });
}
```

> `a.rank` 即 A=14…2=2，V1.2.0 中 `isStraight` 修复时已引入此字段，直接复用。

### 调用位置

在 `render()` 中渲染手牌列表之前，加一行：

```js
const displayHand = sortHand(state.hand);
// 后续用 displayHand 渲染 DOM，state.hand 本身顺序不变（selectedCards 索引基于原始数组）
```

> **重要**：`sortHand()` 返回新数组，不改变 `state.hand` 原始顺序，避免影响 `selectedCards` 的索引引用。

### 新增状态字段

```js
state.sortMode = 'rank';  // 'rank' | 'suit'
```

### 切换按钮事件

```js
document.getElementById('btn-sort-rank').addEventListener('click', () => {
  state.sortMode = 'rank';
  render();
});
document.getElementById('btn-sort-suit').addEventListener('click', () => {
  state.sortMode = 'suit';
  render();
});
```

## 7. 验收 Checklist

- [ ] 发牌后，8 张手牌自动按点数从大到小排列（A > K > Q > J > 10 … > 2）
- [ ] 相同点数时，按 ♠ > ♥ > ♦ > ♣ 排列（示例：K♠ 排在 K♥ 之前）
- [ ] 出牌补牌后，新加入的牌融入排序，手牌保持有序
- [ ] 弃牌补牌后，同上
- [ ] 重新开始后，新一局手牌仍自动排序
- [ ] 点击「按花色」切换后，手牌立即按花色主键重排
- [ ] 点击「按点数」切换回来，手牌立即恢复点数排序
- [ ] 切换按钮激活状态高亮正确（当前模式高亮）
- [ ] 排序不影响已选中牌（选中状态在切换排序后仍然正确保留）
- [ ] `state.hand` 原始数组顺序不被 `sortHand()` 修改

## 8. 不做什么

- **拖拽手动排序**：用户自定义拖拽位置，实现成本高、移动端体验差，且自动排序已解决核心问题
- **自动选牌 / 最优牌型推荐**：超出本轮范围，属于 AI 辅助功能，单独评估
- **跨局持久化排序偏好**：不写 localStorage，每局开始默认「按点数」，足够简单
- **排序动画**：牌位移动的滑动动画，增加实现复杂度，且手牌重排是瞬发操作，不需要动画过渡
- **第三种排序模式**（如「按已选/未选」「按与当前牌型匹配度」）：留后续版本
